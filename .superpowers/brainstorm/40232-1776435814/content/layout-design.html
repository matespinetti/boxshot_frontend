Key Implementation Points

  • AppSidebar — a dedicated components/layout/AppSidebar.tsx client component with usePathname() for active state
  • SidebarProvider wraps the layout — handles Sheet on mobile automatically
  • SidebarTrigger renders as hamburger button on mobile (shadcn handles this)
  • Active state — pathname.startsWith(route) for admin sub-links, exact match for Generate
  • Page stubs — each page returns <PageHeader title="..." /> using the shared component from Section 2
  • loading.tsx — renders a simple full-width skeleton, no per-page customisation
  • error.tsx — client component with reset() button, shows the error message
  • No Zustand needed — shadcn SidebarProvider owns sidebar state internally
